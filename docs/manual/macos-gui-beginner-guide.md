# macOS 图形界面小白使用指南

这份指南面向 macOS 上不想使用终端的普通用户。目标是把安卓导出的照片 / 视频处理成一份安全副本，再由你手动导入 Apple Photos。

PhotoDateRescue 的 GUI 不会修改原始文件，不会自动导入 Apple Photos / iCloud，也不会删除任何照片或视频。

## 适合解决什么问题

- 安卓换 iPhone 后，部分照片跑到“导入当天”。
- 微信、QQ、下载图、截图等外部来源图片在 iPhone 时间线里乱掉。
- 一些视频缺少 Apple Photos 能识别的创建时间。
- 想先生成报告和安全副本，确认没问题后再导入相册。

## 第一步：先把安卓照片 / 视频导出到 Mac

请先把安卓手机里的照片和视频导出到 Mac 上的一个普通文件夹，比如：

```text
/Users/you/Pictures/Android-export
```

推荐尽量完整导出这些目录：

- `DCIM`
- `Pictures`
- `Movies`
- 微信、QQ、下载等你关心的媒体目录

如果你会使用 `adb`，优先使用能保留文件时间的导出方式。不会也没关系，只要先把文件放到 Mac 文件夹里即可。

## 第二步：打开 PhotoDateRescue.app

双击打开 `PhotoDateRescue.app`。

如果 macOS 提示“无法打开，因为无法验证开发者”，这是未签名开源工具常见提示。你可以在确认来源可信后：

1. 右键点击 `PhotoDateRescue.app`。
2. 选择“打开”。
3. 在弹窗里再次选择“打开”。

如果窗口打开后是空白，请让维护者用 `scripts/build-macos-app.sh` 重新构建。这个问题通常是旧版系统 Tcl/Tk 打包导致的。

## 第三步：选择两个文件夹

GUI 里需要选择两个位置：

- 安卓导出文件夹：你刚刚从安卓手机导出的原始照片 / 视频文件夹。
- 安全输出文件夹：PhotoDateRescue 用来写报告和修复副本的新文件夹。

注意：

- 不要把输出文件夹选成原始文件夹。
- 不要直接选 Apple Photos 图库文件。
- 建议输出文件夹放在原始文件夹旁边，比如 `Android-export-PhotoDateRescue-output`。

## 第四步：先扫描

点击“开始扫描”。

扫描完成后，你会看到：

- 共发现多少文件。
- 多少照片、多少视频。
- 多少文件可修复。
- 多少文件正常、风险较高或暂不支持。
- 报告位置。

如果扫描结果和你预期差很多，先不要继续生成副本。请先检查是不是选错了文件夹，或者导出目录不完整。

## 第五步：生成修复后的安全副本

确认扫描结果合理后，点击“生成修复后的安全副本”。

生成完成后，输出文件夹里通常会有：

```text
PhotoDateRescue-output/
  scan-report/
  repaired-media/
```

其中：

- `scan-report/` 是扫描报告。
- `repaired-media/` 是修复后的照片 / 视频副本。

原始文件不会被修改。

## 第六步：先小批量导入 Apple Photos

不要一上来就把全部结果导入 Apple Photos。

建议先从 `repaired-media/` 里挑几十张样本导入 Apple Photos，重点检查：

- 原本跑到导入当天的图片，是否回到正确日期。
- 视频是否出现在合理时间。
- 已经正常的照片是否没有变乱。

确认小样本没问题后，再批量导入更多文件。

## 为什么 GUI 目前不处理 Live Photo

普通照片 / 视频时间线修复，本质是给安全副本写入更合理的时间元数据。失败时也比较容易发现和回滚，因为原图不动。

Live Photo 恢复更复杂，它通常需要：

1. 识别安卓 Motion Photo、MicroVideo 或小红书 sidecar 等不同来源格式。
2. 拆出静态图和短视频。
3. 判断它们是不是同一组动态照片。
4. 调用 macOS 专用工具写入 Apple Photos 识别 Live Photo 所需的配对元数据。
5. 手动导入 Apple Photos 验证是否真的显示为实况照片。

这里最麻烦的是第 4、5 步：工具可能生成了文件，但 Apple Photos 不一定识别成 Live Photo。不同安卓品牌和不同 App 保存的动态照片格式也不完全一样。

所以当前 GUI 先只放普通照片 / 视频修复，避免让新手误以为“点一下就一定恢复所有 Live Photo”。高级动态照片流程仍建议使用 CLI，并先 dry-run 和小样本验证。

## 出问题时先检查什么

- 是否选对了安卓导出文件夹。
- 输出文件夹是否和源文件夹分开。
- `ExifTool` 是否显示“已找到”。
- 视频较多时，`FFmpeg` 是否显示“已找到”。
- 是否先做了小样本导入 Apple Photos 验证。

如果结果不符合预期，请保留：

- 原始导出文件夹。
- 输出文件夹。
- `scan-report/` 里的报告。

不要急着删除原始照片或清空手机相册。
