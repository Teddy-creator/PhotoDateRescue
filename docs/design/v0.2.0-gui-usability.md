# v0.2.0 GUI Usability Design

## Goal

Improve the beginner GUI so users can better understand what to select, what the scan result means, and what to do after repair.

## Scope

This version focuses on ordinary photo / video timeline repair.

In scope:

- Make the auto-suggested output folder more visible.
- Show platform boundaries directly in the GUI.
- Format scan results into clearer categories.
- Show safer next-step guidance after repair.
- Keep macOS and Windows GUI behavior aligned where possible.

Out of scope:

- Windows `.exe` packaging.
- Live Photo / Motion Photo GUI recovery.
- Automatic Apple Photos / iCloud import.
- Deleting, deduplicating, or modifying source media.
- Large visual redesign or custom UI framework migration.

## UX Rules

- The GUI should continue to feel conservative and safe.
- Every success message should tell the user the next safe action.
- Every platform claim must preserve the current boundary: Windows supports ordinary photo / video repair, not Apple Photos-recognized Live Photo generation.
- Output folder suggestions should be helpful but never silently overwrite a user-chosen output folder.

## Implementation Notes

- Keep the current Tkinter app and controller split.
- Add small pure-formatting helpers that can be tested without launching a window.
- Avoid introducing new dependencies.
- Update README and GUI beginner docs only where behavior changes.
