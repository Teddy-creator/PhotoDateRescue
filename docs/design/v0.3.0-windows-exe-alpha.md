# v0.3.0 Windows EXE Alpha Design

## Goal

Produce an experimental Windows GUI executable build through GitHub Actions so Windows users can try PhotoDateRescue without running Python commands manually.

## Scope

In scope:

- Build a Windows GUI executable from `photodaterescue.gui_launcher`.
- Package the executable output as a zip artifact.
- Document the alpha boundary and dependency requirements.
- Keep normal photo / video timeline repair as the only GUI promise.

Out of scope:

- Code signing.
- Windows installer creation.
- Bundling ExifTool, FFmpeg, or adb.
- Apple Photos-recognized Live Photo generation on Windows.
- Automatic imports, deletion, deduplication, or cloud upload.

## Packaging Approach

The alpha build uses GitHub Actions on `windows-latest` with Python 3.12 and PyInstaller. The workflow builds from the current source tree, runs tests first, then packages the GUI launcher as `PhotoDateRescue.exe`.

For tag builds, the workflow also uploads the generated zip to the GitHub Release. Manual workflow runs still publish the zip as an Actions artifact only.

External tools remain user-installed:

- `exiftool.exe` is required at runtime.
- `ffmpeg.exe` / `ffprobe.exe` are recommended for video handling.
- `adb.exe` is optional for Android export flows.

## Risks

- Unsigned `.exe` files may trigger Windows SmartScreen or antivirus warnings.
- The build artifact is an alpha convenience package, not a polished installer.
- Real Windows machine validation is still required before calling the `.exe` stable.
